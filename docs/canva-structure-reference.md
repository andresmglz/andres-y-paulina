# Canva Structure Reference

Fuente revisada el 26 de marzo de 2026:
https://andresandpaulina.my.canva.site

Objetivo de esta nota:
- Reutilizar la organizacion de informacion del sitio de Canva.
- No copiar estilos, colores ni copy placeholder.
- Traducir esa estructura a la landing editorial de este repo.

## Orden de secciones detectado

1. Portada
- Nombres de los novios en grande.
- Fecha del evento.
- Hashtag o frase corta.

2. Historia
- Un bloque narrativo principal sobre la relacion.

3. Logistica resumida
- Ceremonia.
- Codigo de vestimenta.
- Post-ceremonia o recepcion.

4. Timeline del dia
- Horarios concretos por momento del evento.

5. Regalos
- Mensaje principal.
- Instruccion de contacto o metodo de regalo.

6. FAQ
- Preguntas practicas.
- Accesos, acompanantes, ninos, estacionamiento, llegada.

7. RSVP / contacto
- Cierre con personas de contacto.
- Telefonos y correos.

## Traduccion recomendada para esta app

- `Hero`: portada con nombres, fecha, ciudad y CTA.
- `Story`: historia real de ustedes.
- `EventDetails`: ceremonia, recepcion y dress code.
- `Timeline`: agenda del dia.
- `Gifts`: mesa de regalos o transferencia.
- `FAQ`: preguntas practicas para invitados.
- `InvitationFlow` o `RSVP bridge`: explicacion de que la confirmacion personalizada llegara por link privado.
- `Footer`: cierre emocional.

## Notas

- El sitio de Canva usa mucho placeholder en ingles.
- La estructura si es aprovechable para `AYP-002`.
- Las fotos de ustedes deben vivir en `public/images/couple/`.
